NuHeat API Reference

Dashboard Interactive Docs
Control your NuHeat floor heating thermostats from any system on your network. Three interfaces are available: a REST API with JSON bodies, a Query String API using simple GET requests, and a Health endpoint for monitoring. For interactive testing, use the Swagger UI.

Rate Limiting & Caching

All read endpoints serve from an in-memory cache that is refreshed from NuHeat every 5 minutes by a background poller. No client request ever hits NuHeat directly for reads. Responses include a last_updated timestamp so you know the data freshness.

Rate limits per IP address: 60 reads/minute, 10 writes/minute. Exceeding the limit returns HTTP 429.

REST API

GET /api/thermostats
List all thermostats. Served from cache.
Example
curl http://localhost:8777/api/thermostats
GET /api/thermostats/{serial_number}
Get a single thermostat's current status.
Path Parameters
NameTypeDescription
serial_numberstring requiredThermostat serial number
Example
curl http://localhost:8777/api/thermostats/ABC123
PUT /api/thermostats/{serial_number}/temperature
Set the target temperature. Provide temperature_c or temperature_f. Omit hold_until for a permanent hold, or provide an ISO datetime for a temporary hold.
Path Parameters
NameTypeDescription
serial_numberstring requiredThermostat serial number
JSON Body
NameTypeDescription
temperature_cfloat optionalTarget temperature in Celsius
temperature_ffloat optionalTarget temperature in Fahrenheit
hold_untilstring optionalISO datetime for temporary hold
Examples
# Set to 72°F with permanent hold
curl -X PUT http://localhost:8777/api/thermostats/ABC123/temperature \
  -H "Content-Type: application/json" \
  -d '{"temperature_f": 72}'

# Set to 25°C until a specific time
curl -X PUT http://localhost:8777/api/thermostats/ABC123/temperature \
  -H "Content-Type: application/json" \
  -d '{"temperature_c": 25, "hold_until": "2025-12-01T08:00:00"}'
POST /api/thermostats/{serial_number}/resume
Resume the programmed schedule for a thermostat.
Example
curl -X POST http://localhost:8777/api/thermostats/ABC123/resume
POST /api/refresh
Force a cache refresh from NuHeat. Throttled to once per 60 seconds. Use this if you need fresher data than the 5-minute polling cycle provides.
Example
curl -X POST http://localhost:8777/api/refresh
GET /api/health
Health check. Returns server status, cached thermostat count, and rate limit config.
Example
curl http://localhost:8777/api/health

Query String API

Simple GET-only endpoints for systems that can't send JSON bodies — webhooks, browser bookmarks, curl one-liners, IoT devices.
GET /qs/status
Get thermostat status from cache. Omit serial for all thermostats.
Query Parameters
NameTypeDescription
serialstring optionalThermostat serial number (omit for all)
Examples
# All thermostats
curl http://localhost:8777/qs/status

# Single thermostat
curl http://localhost:8777/qs/status?serial=ABC123
GET /qs/set
Set thermostat temperature. Provide temp_c or temp_f.
Query Parameters
NameTypeDescription
serialstring requiredThermostat serial number
temp_cfloat optionalTarget temperature in Celsius
temp_ffloat optionalTarget temperature in Fahrenheit
hold_untilstring optionalISO datetime for temporary hold
Examples
# Set to 72°F
curl "http://localhost:8777/qs/set?serial=ABC123&temp_f=72"

# Set to 25°C with temporary hold
curl "http://localhost:8777/qs/set?serial=ABC123&temp_c=25&hold_until=2025-12-01T08:00:00"
GET /qs/resume
Resume the programmed schedule.
Query Parameters
NameTypeDescription
serialstring requiredThermostat serial number
Example
curl "http://localhost:8777/qs/resume?serial=ABC123"
